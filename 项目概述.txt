                     Szwego 商品组装器 - 项目概述
================================================================================

一、项目简介
──────────────────────────────────────────────────────────────────────
本项目是一个基于 Electron 的桌面应用，用于从 szwego.com（微信私域电商平台）
抓取商品动态数据，通过人工筛选 + AI 辅助处理，批量生成符合目标系统导入格式
的商品表格。

核心流程：
  1. 从 szwego.com 抓取商品相册动态（图片+文案）
  2. 人工浏览选择多条动态，组装成一件完整商品
  3. 调用 DeepSeek AI 自动填充商品名称、关键词、分类、详情等字段
  4. 按目标模板格式导出 Excel，支持导入到其他电商系统

二、技术架构
──────────────────────────────────────────────────────────────────────
运行环境：Node.js + Electron 33 + Python 3.12
核心依赖：electron, openpyxl (Python)
AI 接口：DeepSeek API (api.deepseek.com/v1/chat/completions)

进程模型：
  ┌─────────────────────────────────────────────┐
  │ Main Process (main.js)                      │
  │  - 加载 74MB all_items.json 到内存          │
  │  - IPC 分页查询 (200条/页)                  │
  │  - 后台窗口刷新数据（带 Cookie 持久化）     │
  │  - AI 调用 (HTTPS → api.deepseek.com)       │
  │  - 文件读写 (saved_products.json 等)        │
  ├─────────────────────────────────────────────┤
  │ Preload (preload.js)                        │
  │  - contextBridge 暴露 api 对象              │
  │  - 5 个 IPC 通道                            │
  ├─────────────────────────────────────────────┤
  │ Renderer (index.html + builder.js)          │
  │  - 瀑布流卡片浏览                           │
  │  - 多选→自动合并→保存                       │
  │  - AI 配置面板（Tab 切换）                  │
  │  - 工作日志与进度恢复                       │
  └─────────────────────────────────────────────┘

外部脚本：
  export_template.py - Python 独立脚本，导出生效无需重启应用

四、功能模块
──────────────────────────────────────────────────────────────────────

【1. 素材浏览】
  - 22,427条商品动态卡片，220px固定宽度瀑布流
  - 按日期分组（最新在前），蓝色分隔标题
  - 搜索过滤：标题关键词 + 日期 + 有无图片
  - 分页加载：首次200条，点击"加载更多"继续
  - 多图标记：左上角黄色数字显示图片数量
  - 状态标记：右下角绿色✓=已保存，蓝色AI=已AI处理

【2. 商品组装】
  - 点击卡片选中/取消，右侧面板实时更新
  - 自动合并：选中即自动组装，无需手动点合并按钮
  - 图片排列：按选中顺序，自动去重
  - 款号提取：自动识别标题中 #XXXXX 五位编号
  - 价格提取：自动读取 itemPrice 字段
  - 文案拼接：选中素材标题按顺序拼接

【3. AI 处理 (DeepSeek)】
  - 配置区独立 Tab：API Key + 模型选择 + 测试连接
  - 9个AI生成字段，每个带独立开关和自定义提示词模板
    字段：商品名称、副标题、短标题、标题标签、搜索关键词、
          商品特征标签、商品分类、商品规格、移动端详情
  - 提示词模板变量：{copy}=文案, {style}=款号, {price}=价格, {imgs_count}=图片数
  - 并行处理：3个商品同时处理，每商品内字段并行
  - 重试机制：每字段最多重试3次，遇空自动重试（间隔0.8秒）
  - 已处理标记：处理过的商品不再重复处理

【4. 导出 Excel】
  - 严格按 shitimoban.xlsx 31列模板格式
  - 每商品生成3行（S/M/L尺码拆分）
  - 硬编码默认值一览：
    ┌────────────┬────────────────────────────┐
    │ 商品支持    │ 承诺包退;厂家直发;假一赔四 │
    │ 库存        │ 1000                       │
    │ 重量        │ 0.3 (千克)                 │
    │ 会员权益    │ 会员价;会员等级折扣        │
    │ 显示库存    │ 隐藏                       │
    │ 显示销量    │ 显示                       │
    │ 物流支持    │ 快递发货;同城配送;上门自提  │
    │ 配送方式    │ 快递发货                   │
    │ 快递运费    │ 运费模板:2234              │
    │ 商品状态    │ 上架售卖                   │
    │ 出售渠道    │ 公众号;小程序;PC;抖音;H5   │
    │ 商品规格    │ [颜色:图片色][尺码:S/M/L]  │
    │ 商品分组    │ 2026                       │
    │ 划线价      │ 原始价×2.3→四舍五入→9结尾 │
    │ 售卖价      │ 原始价×2                   │
    │ 商品图片    │ 前5张（轮播图限制）        │
    │ 移动端详情  │ 全部图片（不受5张限制）    │
    └────────────┴────────────────────────────┘

【5. 数据刷新】
  - 从 szwego API 拉取近7天数据
  - 后台隐藏窗口加载 szwego 页面获取有效 Cookie
  - 从渲染进程 context 发 fetch（自动带 Cookie）
  - 频率控制：每页600ms间隔，最多20页
  - 增量合并：按 goods_id 去重

【6. 工作日志与恢复】
  - 自动保存工作状态（选中的素材、已保存商品ID、已处理ID）
  - 重启后恢复标记（✓已保存、AI已处理）
  - 操作日志记录（启动/关闭/保存/导出/刷新）
  - 日志上限500条

【7. 登录管理】
  - Cookie 持久化：partition: 'persist:szwego'
  - 登录过期时弹出 szwego 登录窗口
  - 图片CDN防盗链：自动添加 Referer 请求头

五、IPC 接口一览
──────────────────────────────────────────────────────────────────────
渲染进程 → 主进程 (通过 preload.js 暴露的 window.api)：

  api.page(filter)        分页查询商品（search/date/img/offset/limit）
  api.dates()             获取所有日期列表
  api.save(json)          保存已组装商品到 saved_products.json
  api.loadSaved()         加载已保存商品
  api.saveState(json)     保存工作状态到 work_state.json
  api.loadState()         加载工作状态
  api.log(action, detail) 写入操作日志
  api.getLog()            读取操作日志
  api.export(json)        导出Excel（调用Python脚本）
  api.fetch()             从szwego刷新数据
  api.login()             打开登录窗口
  api.aiSaveConfig(c)     保存AI配置（含提示词模板和开关）
  api.aiLoadConfig()      加载AI配置
  api.aiTest(c)           测试AI连接
  api.aiChat(c, m)        AI对话（通用接口）
  api.aiProcess(cfg, prods, prompts, enabled)  批量AI处理
  api.onAiProgress(cb)    监听AI处理进度
  api.onLoginDone(cb)     监听登录完成

六、数据流
──────────────────────────────────────────────────────────────────────

抓取阶段：
  szwego API (/album/personal/all)
    → all_items.json (22,427条, 74MB)
    → 主进程内存 (cachedItems)
    → IPC分页 (200条/页)
    → 渲染进程显示

组装阶段：
  选中素材 → selectedCache → autoMerge()
    → 提取款号 (#\d{5}) + 价格 (itemPrice)
    → 拼接文案 + 合并图片
    → 保存 → saved_products.json

AI处理阶段：
  已保存商品 → aiProcess()
    → 并行调用 DeepSeek (每字段独立重试)
    → 结果存入内存 (aiResults)
    → 标记 processedIds

导出阶段：
  已保存商品 + AI结果
    → export_template.py (Python)
    → 严格按照 shitimoban.xlsx 31列格式
    → 每商品3行 (S/M/L)
    → exports/products_xxx.xlsx

  4. 数据刷新失败 → Cookie可能过期，先点"重新登录"再刷新
  5. 导出失败 → 检查 Python 是否安装 openpyxl
               检查模板文件 C:\Users\qingf\Desktop\下载\shitimoban.xlsx 是否存在
